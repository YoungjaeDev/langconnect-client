# MCP Server Setup for WSL Environment

This guide explains how to configure LangConnect MCP server when running in WSL and connecting from Windows Claude Desktop or Cursor.

## Overview

LangConnect provides two MCP server modes:
- **stdio mode**: Direct process communication (recommended for Claude Desktop)
- **SSE mode**: HTTP-based server (for testing and multi-client scenarios)

This document focuses on stdio mode configuration for WSL environments.

## Prerequisites

- WSL2 installed and configured
- LangConnect project set up in WSL at `/home/yeong/workspace/langconnect-client`
- Docker services running (`make up`)
- Python virtual environment created (`uv sync`)
- Valid Supabase account and credentials

## Quick Start

### 1. Generate MCP Configuration

In WSL terminal:

```bash
cd /home/yeong/workspace/langconnect-client
make mcp
```

This will:
- Prompt for your Supabase email and password
- Authenticate and obtain access token
- Generate `mcpserver/mcp_config.json` with your credentials

### 2. Configure Claude Desktop (Windows)

Open the Claude Desktop configuration file:

```
%APPDATA%\Claude\claude_desktop_config.json
```

Add the following configuration (replace paths and tokens with your actual values):

```json
{
  "mcpServers": {
    "langconnect-rag-mcp": {
      "command": "wsl.exe",
      "args": [
        "bash",
        "-c",
        "/home/yeong/.local/bin/uv --directory /home/yeong/workspace/langconnect-client/mcpserver run mcp_server.py"
      ],
      "env": {
        "API_BASE_URL": "http://localhost:8080",
        "SUPABASE_JWT_SECRET": "your-jwt-token-from-make-mcp",
        "OPENAI_API_KEY": "your-openai-api-key"
      }
    }
  }
}
```

### 3. Restart Claude Desktop

Completely quit and restart Claude Desktop (not just close the window).

### 4. Verify Connection

Look for the hammer icon in Claude Desktop's chat input area. Click it to see available MCP tools:
- `search_documents`
- `list_collections`
- `create_collection`
- `add_documents`
- And more...

## Configuration Options

### Option 1: Using uv (Recommended)

```json
{
  "command": "wsl.exe",
  "args": [
    "bash",
    "-c",
    "/home/yeong/.local/bin/uv --directory /home/yeong/workspace/langconnect-client/mcpserver run mcp_server.py"
  ]
}
```

Advantages:
- Automatic dependency management
- Cleaner project structure
- Built-in virtual environment handling

### Option 2: Using Virtual Environment Python Directly

```json
{
  "command": "wsl.exe",
  "args": [
    "bash",
    "-c",
    "/home/yeong/workspace/langconnect-client/.venv/bin/python /home/yeong/workspace/langconnect-client/mcpserver/mcp_server.py"
  ]
}
```

Advantages:
- Direct execution
- No additional tools required
- Explicit path control

### Option 3: With Working Directory Change

```json
{
  "command": "wsl.exe",
  "args": [
    "bash",
    "-c",
    "cd /home/yeong/workspace/langconnect-client && .venv/bin/python mcpserver/mcp_server.py"
  ]
}
```

Advantages:
- Ensures correct working directory
- Useful for relative imports
- Simpler path specifications in server code

## Path Mapping Reference

### WSL to Windows Path Mapping

| Context | Path Type | Example |
|---------|-----------|---------|
| Claude Desktop Config | WSL absolute paths | `/home/yeong/workspace/project` |
| WSL accessing Windows files | `/mnt/c/...` | `/mnt/c/Users/username/Documents` |
| Windows accessing WSL files | `\\wsl.localhost\...` | `\\wsl.localhost\Ubuntu\home\yeong` |

### Important Path Rules

1. Always use WSL absolute paths in configuration
2. Never use Windows-style paths (`C:\...`) in WSL args
3. Environment variables are evaluated in WSL context
4. Working directory defaults to Windows user home unless specified

## Environment Variables

### Required Variables

```json
"env": {
  "API_BASE_URL": "http://localhost:8080",
  "SUPABASE_JWT_SECRET": "your-jwt-token",
  "OPENAI_API_KEY": "your-openai-key"
}
```

### Variable Sources

- `API_BASE_URL`: LangConnect API endpoint (default: http://localhost:8080)
- `SUPABASE_JWT_SECRET`: Generated by `make mcp` (expires in ~1 hour)
- `OPENAI_API_KEY`: Your OpenAI API key from `.env` file

### Alternative: Inline Environment Variables

If the `env` property doesn't work, you can pass variables inline:

```json
{
  "args": [
    "bash",
    "-c",
    "API_BASE_URL=http://localhost:8080 SUPABASE_JWT_SECRET=eyJ... /path/to/python /path/to/mcp_server.py"
  ]
}
```

## Troubleshooting

### Authentication Issues

**Problem**: "401 Unauthorized" errors in MCP tools

**Cause**: JWT token expired (tokens last ~1 hour)

**Solution**:
```bash
cd /home/yeong/workspace/langconnect-client
make mcp  # Re-authenticate and generate new token
```

Then update the `SUPABASE_JWT_SECRET` in your Claude Desktop config and restart.

### Connection Issues

**Problem**: MCP server doesn't appear in Claude Desktop

**Checklist**:
1. Verify WSL is installed: `wsl --version`
2. Check API server is running: `curl http://localhost:8080/health`
3. Test uv installation: `wsl which uv`
4. Verify Python path: `wsl ls /home/yeong/workspace/langconnect-client/.venv/bin/python`
5. Check Claude Desktop logs: `%APPDATA%\Claude\logs\mcp-server-langconnect-rag-mcp.log`

### Path Issues

**Problem**: "File not found" or "Module not found" errors

**Solutions**:
- Use absolute paths in WSL format
- Verify paths exist: `wsl ls -la /home/yeong/workspace/langconnect-client/mcpserver`
- Check working directory with `cd` in command
- Ensure virtual environment is activated

### Command Testing

Test the MCP server command manually in WSL:

```bash
# Test 1: Check uv works
/home/yeong/.local/bin/uv --version

# Test 2: Run server directly
cd /home/yeong/workspace/langconnect-client
API_BASE_URL=http://localhost:8080 \
SUPABASE_JWT_SECRET=your-token \
OPENAI_API_KEY=your-key \
uv run python mcpserver/mcp_server.py

# Test 3: Test from Windows
wsl bash -c "/home/yeong/.local/bin/uv --directory /home/yeong/workspace/langconnect-client/mcpserver run mcp_server.py"
```

### Log Analysis

**Windows Claude Desktop logs location**:
```
%APPDATA%\Claude\logs\mcp.log
%APPDATA%\Claude\logs\mcp-server-langconnect-rag-mcp.log
```

**Common error patterns**:

| Error Message | Cause | Solution |
|--------------|-------|----------|
| `spawn wsl.exe ENOENT` | WSL not in PATH | Reinstall WSL or add to PATH |
| `UNC paths not supported` | Wrong path format | Use `/mnt/c/...` not `\\wsl$\...` |
| Connection closed immediately | Server crashed | Check stderr logs |
| No tools appear | Config not loaded | Fully restart Claude Desktop |
| Import errors | Wrong working directory | Add `cd` to command |

## Advanced Configuration

### SSE Mode (Alternative)

For development and testing, you can use SSE mode instead:

**1. Start SSE server in WSL**:
```bash
cd /home/yeong/workspace/langconnect-client
./run_mcp_sse.sh
```

**2. Configure Claude Desktop**:
```json
{
  "mcpServers": {
    "langconnect-rag-mcp": {
      "url": "http://localhost:8765/sse",
      "transport": "sse"
    }
  }
}
```

**Note**: SSE mode is not officially supported by Claude Desktop as of 2025, but can be tested with MCP Inspector:

```bash
npx @modelcontextprotocol/inspector
# Select "SSE" transport, URL: http://localhost:8765/sse
```

### Network Configuration

For SSE mode to work reliably in WSL2, configure mirrored networking:

**File**: `%USERPROFILE%\.wslconfig`

```ini
[wsl2]
networkingMode = mirrored
```

Then restart WSL:
```powershell
wsl --shutdown
```

### Long-Running Server

To keep the SSE server running in the background:

```bash
# In WSL
cd /home/yeong/workspace/langconnect-client
nohup ./run_mcp_sse.sh > mcp-server.log 2>&1 &

# Check if running
ps aux | grep mcp_sse_server

# Stop server
pkill -f mcp_sse_server.py
```

## Security Considerations

### Token Management

1. Never commit `mcp_config.json` to version control
2. JWT tokens expire after 1 hour for security
3. Store OpenAI API key in `.env` file (gitignored)
4. Use environment variables in Claude config, not hardcoded values

### Access Control

- MCP server uses your Supabase user authentication
- All operations are scoped to your user ID
- API server validates tokens on every request
- Multi-tenancy ensures data isolation

## Best Practices

1. **Keep tokens fresh**: Regenerate tokens regularly or implement refresh logic
2. **Use absolute paths**: Avoid relative paths that depend on working directory
3. **Check logs**: Monitor `%APPDATA%\Claude\logs\` for issues
4. **Test independently**: Verify server works before connecting to Claude
5. **Restart cleanly**: Fully quit Claude Desktop after config changes
6. **Document changes**: Keep track of customizations for team sharing

## Available MCP Tools

Once configured, you'll have access to these tools in Claude Desktop:

### Document Search
- `search_documents`: Semantic, keyword, or hybrid search
- `multi_query`: RAG query expansion for better results

### Collection Management
- `list_collections`: View all your collections
- `get_collection`: Get details of a specific collection
- `create_collection`: Create a new collection
- `delete_collection`: Remove a collection and its documents

### Document Management
- `list_documents`: View documents in a collection
- `add_documents`: Add text documents to a collection
- `delete_document`: Remove a document from a collection

### System
- `get_health_status`: Check API server status
